# План выполнения тестового задания "Telegram-чат"

## Что понятно сразу
*   Цель: создать однопользовательский чат, похожий на Telegram, с сохранением истории сообщений.
*   Стек: React + TypeScript (фронтенд), Python + FastAPI (бэкенд), деплой на публичный сервер.
*   Основной функционал: отправка, получение и сохранение сообщений.

## Что требовало уточнения
*   Дизайн точных цветов и отступов Telegram (решила искать скриншоты и адаптировать).
*   Структура API (решила сделать два простых эндпоинта: GET и POST `/messages`).
*   Выбор сервисов для деплоя (остановилась на Vercel для фронтенда и Render для бэкенда).

## Порядок выполнения
1.  **Настройка проекта**: создание репозитория, инициализация React и FastAPI приложений.
2.  **Разработка бэкенда**: создание API для работы с сообщениями, настройка CORS.
3.  **Разработка фронтенда**: создание интерфейса чата, стилизация под Telegram, подключение к API.
4.  **Интеграция и тестирование**: проверка полного цикла работы приложения.
5.  **Деплой**: публикация фронтенда и бэкенда на выбранных сервисах.
6.  **Документация**: написание этого плана и итогового саморевью.

## Принятые решения и их обоснование
*   **FastAPI для бэкенда**: Быстрый старт, автоматическая документация Swagger, простота.
*   **Хранение в оперативной памяти (с последующим переходом на SQLite)**: Соответствует минимальным требованиям задания, но показывает умение работать с БД.
*   **Простой API дизайн (GET/POST `/messages`)**: Полностью покрывает требуемый функционал, понятен и легко тестируется.
*   **Деплой на Vercel и Render**: Бесплатные и удобные сервисы, подходящие для демонстрационных проектов.
