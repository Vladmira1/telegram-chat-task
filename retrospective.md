# Саморевью и ретроспектива

## Что получилось хорошо
*   **Рабочий прототип**: Основной функционал (отправка, получение, сохранение сообщений) реализован и работает корректно.
*   **Внешний вид**: Интерфейс стилизован под Telegram, использованы характерные цвета и расположение элементов.
*   **Чистая структура проекта**: Код разделён на фронтенд и бэкенд, коммиты логично отражают историю разработки.

## С какими сложностями столкнулась
*   **Настройка окружения**: Первоначальные трудности с установкой Node.js, Git и настройкой виртуального окружения Python.
*   **Работа с Git/GitHub**: Первый опыт создания репозитория, клонирования и отправки коммитов требовал внимательности к командам.
*   **Понимание CORS**: Потребовалось время, чтобы разобраться, как правильно настроить бэкенд для приёма запросов с фронтенда.

## Что можно было сделать иначе с большим временем
*   Более детально проработать UI/UX: добавить анимации отправки сообщений, статус "печатает..." (typing indicator).
*   Реализовать полноценную авторизацию для многопользовательского режима.
*   Написать unit-тесты для ключевых функций бэкенда и компонентов фронтенда.
*   Использовать state-менеджер (например, Redux Toolkit) на фронтенде для более сложной логики.

## Возможные улучшения
*   **Переход с памяти на базу данных**: Реализация хранения в SQLite или PostgreSQL для персистентности данных.
*   **Оптимизация обновлений чата**: Замена периодического опроса (polling) на WebSocket для мгновенного получения новых сообщений.
*   **Улучшение обработки ошибок**: Более информативные сообщения об ошибках для пользователя и логирование на бэкенде.
