# Саморевью и ретроспектива

## Что получилось хорошо
*   **Полностью рабочее приложение**: Основной функционал (отправка, получение, сохранение сообщений) реализован. Проект задеплоен, для проверки не требуется локальный запуск.
*   **Архитектура и дизайн**: Код разделён на фронтенд (React/TypeScript) и бэкенд (Python/FastAPI). Интерфейс стилизован под светлую тему Telegram. Использована база данных SQLite для надёжного хранения.
*   **Работа с системой контроля версий**: Коммиты в Git логично отражают этапы разработки: настройку, создание функционала, исправление ошибок, деплой.

## С какими сложностями столкнулась
*   **Настройка окружения и деплоя**: Наибольшее время заняла настройка деплоя на новых сервисах (Render, Netlify), работа с CORS и корректная конфигурация команд сборки и путей.
*   **Понимание CORS**: Потребовалось разобраться, как правильно настроить бэкенд для приёма запросов с определённого домена фронтенда и как диагностировать эту ошибку.
*   **Работа с Git/GitHub**: Как первый проект, потребовал внимательности к командам добавления файлов и отправки изменений.

## Что можно было сделать иначе с большим временем
*   **Улучшить UX**: Добавить анимации, индикатор отправки сообщения, звуковое уведомление.
*   **Повысить надёжность**: Реализовать более строгую обработку сетевых ошибок на фронтенде, добавить валидацию данных на бэкенде.
*   **Добавить функции**: Реализовать отправку изображений, отметки о прочтении, возможность редактирования сообщений.
*   **Автоматизировать тестирование**: Написать end-to-end тесты для проверки полного цикла работы приложения.

## Возможные улучшения текущей реализации
*   **Настройка CORS на бэкенде**: Вместо временного прокси окончательно настроить правильные заголовки CORS на сервере Render.
*   **Оптимизация производительности**: Кэширование запросов на фронтенде, настройка пагинации для получения сообщений.
*   **Улучшение безопасности**: Использование переменных окружения для хранения чувствительных данных (например, если бы были ключи API).

**Важное примечание:**
Бэкенд размещён на бесплатном тарифе Render.com, который переходит в спящий режим после 15 минут бездействия. 
Первый запрос после простоя может занимать **30-60 секунд** на запуск сервиса.

Для немедленной проверки можно предварительно "разбудить" сервис, открыв:
- https://telegram-chat-backend-9lwd.onrender.com
- https://telegram-chat-backend-9lwd.onrender.com/docs

После этого чат будет работать без задержек.
